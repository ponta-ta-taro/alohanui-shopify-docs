## 開発ワークフロー

### 初回セットアップ
```bash
cd C:\projects\alohanui-demo
shopify theme dev --store alohanui-demo
```

### 日常的な開発フロー
1. **開始前の確認**
   - `HANDOVER.md` で現在の進捗を確認
   - `docs/prompts/` から該当プロンプトを読み込む

2. **開発サーバー起動**
   ```bash
   shopify theme dev
   ```

3. **実装作業**
   - プロンプトに従って実装
   - ローカルでプレビュー確認

4. **記録**
   - `docs/implementation-log.md` に作業内容を記録
   - 気づきや改善点をメモ

5. **テスト環境へデプロイ**
   ```bash
   shopify theme push --store alohanui-demo
   ```

### 本番デプロイ（慎重に）
```bash
# テスト完了後のみ
shopify theme push --store alohanui-jp --unpublished
# 管理画面で手動公開
```

## 重要な決定事項

### 開発方針
1. **段階的リリース**: Phase 1から順次実装し、効果を確認しながら進める
2. **本番は触らない**: 必ず開発ストアで検証してから本番反映
3. **レスポンシブファースト**: モバイル体験を最優先
4. **パフォーマンス重視**: 画像最適化、コード軽量化

### 技術選定
- **テーマ**: Dawn（Shopify公式）をベースにカスタマイズ
- **アプリ**: 必要最小限（Judge.meなど）
- **コーディング規約**: BEM方式、セクションベース開発

### コミュニケーション
- 重要な変更は `docs/decisions.md` に記録
- 実装前にプロンプトで方針確認
- 困ったら `HANDOVER.md` に戻る

## トラブルシューティング

### Shopify CLIが動かない
```bash
shopify version
# バージョン確認。古い場合は更新
npm update -g @shopify/cli @shopify/theme
```

### ローカルプレビューが表示されない
- ストア名が正しいか確認
- `shopify auth logout` してログインし直す

### 本番を誤って触りそうになった
- **落ち着いて**: まず一旦停止
- ストア名を必ず確認：`alohanui-demo` か？
